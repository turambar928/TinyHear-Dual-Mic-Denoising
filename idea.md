# 边缘端侧（M55/U55）环境噪声抑制与人声分离技术调研

---

## 一、 核心技术原理与方法分类

在边缘端侧（Endpoint AI）芯片上，音频处理算法通常在**传统信号处理（DSP）**、**纯深度学习（DL）**以及**混合方案（Hybrid）**之间进行权衡。其核心目标是在复杂声学环境中提取目标人声并抑制背景噪声。

### 1. 传统信号处理方法（基于统计与物理特性）

此类方法无需数据训练，内存开销极小（K级），计算确定性高，但对非平稳噪声（如突发敲击声、背景干扰人声）的抑制能力有限。

* **谱减法（Spectral Subtraction）**：假设噪声在短时间内是平稳的，在无说话声的静音期（通过语音激活检测 VAD 判断）估计噪声功率谱，并在带噪信号中将其减去。
* **维纳滤波（Wiener Filtering）**：基于均方误差最小化准则，利用信噪比（SNR）动态估计增益，对信号进行平滑滤波和增益控制。
* **多麦克风波束成形（Beamforming）**：结合物理空间双麦或多麦阵列，利用到达时间差（TDOA）形成具有指向性的空间蜂窝。**在助听器领域，这是对抗“鸡尾酒会效应”的第一道防线**，能天然抑制非目标方向的干扰。

### 2. 深度学习方法（基于特征学习）

深度学习将音频降噪与分离抽象为**时频域掩蔽（Masking）**或**时域映射（Mapping）**问题：

* **时频域掩蔽（T-F Masking）**：将输入波形通过短时傅里叶变换（STFT）转换为时频图（Spectrogram）。神经网络预测一个掩蔽矩阵（如理想比值掩蔽 IRM 或复数掩蔽 CRM），与原时频图相乘后再通过逆短时傅里叶变换（ISTFT）恢复。该方案特征维度低，极适合端侧。
* **时域端到端分离**：绕过 STFT，直接通过一维卷积（Encoder）将时域波形映射到高维隐空间进行分离，再通过 Decoder 重构。虽然性能优异，但隐空间通道数通常较大，对端侧 SRAM 内存构成巨大挑战。

## 二、 适合 M55/U55 芯片的轻量化微模型架构

Arm Cortex-M55（支持 Helium 向量扩展指令集）与 Ethos-U55（微型 NPU）提供了 MCU 级别的专用 ML 算力。适合该架构的模型必须具备**小通道数、高重用性、纯因库（Causal）结构**。

### 1. 心理声学驱动的 RNN/GRU 架构（如 RNNoise）

* **原理**：神经网络不直接处理几万个传统的 FFT 频率点，而是处理经过梅尔刻度（Mel-scale）或巴克刻度（Bark-scale）划分的少量频带（如 22 或 32 个临界频带）。
* **典型模型**：**RNNoise**（由 Xiph.Org 开源）。
* **端侧优势**：模型通常仅包含一个轻量全连接与几层 GRU，参数量极小（$< 100 \text{ KB}$），运行内存消耗微乎其微。M55 的 Helium 指令集或 U55 可以极其高效地流水线加速此类全连接和循环算子。

### 2. 轻量化复数卷积网络（Tiny DCCRN / DCUNet）

* **原理**：传统网络只预测幅度谱，丢弃了相位信息，导致重构音质粗糙。DCCRN 引入**复数卷积（Complex Convolution）**，同时处理音频的实部（Real）和虚部（Imaginary），对相位的恢复极好。
* **典型模型**：裁剪版 DCCRN（Deep Complex Convolutional Recurrent Network）。
* **端侧优势**：将复数运算拆解为两条实数路径（$(A+jB)(C+jD) = (AC-BD) + j(BC+AD)$），可完美转换为 U55 原生支持的普通 2D 卷积和矩阵减法，在保证音质的同时实现硬件满速加速。

### 3. 多速率子带神经网络（Sub-band NN）

* **原理**：利用多相滤波器组（Polyphase Filterbank）将音频切分为多个子带（如 4 或 8 个子带）。对于低频和人声核心区，使用 U55 运行微型神经网络预测增益；对于高频区，直接使用传统轻量算法。
* **端侧优势**：这种非对称的多速率处理能大幅省下不必要的算力和功耗。

## 三、 助听器（Hearing Aids）场景下的极端工程约束

在将算法应用于助听器研究时，各项技术指标会受到比普通 TWS 耳机严苛数倍的限制：

| 约束维度     | 工业级指标要求                                       | 痛点与端侧技术对策                                           |
| :----------- | :--------------------------------------------------- | :----------------------------------------------------------- |
| **超低延迟** | **$< 8\text{ ms}$** （甚至更苛刻的 $< 5\text{ ms}$） | 听障患者能听到穿透耳道的物理漏音。若电信号延迟过大，会产生明显的**梳状滤波器效应**引发头晕。**对策**：STFT 帧进（Hop Size）必须缩短至 $2\text{ms}$-$4\text{ms}$，算法拒绝任何未来帧（Look-ahead）。 |
| **高保真度** | 高 HASQI / HASPI 指标                                | 强烈的机械音和音门瞬变易导致听觉疲劳。**对策**：AI 应预测**软掩蔽（Soft Mask）**，并设立最小增益下限（如仅压制 12-18dB 噪声），保留环境空间感知。 |
| **极低功耗** | **$< 1\text{ mW}$** (整个音频 DSP/AI 链)             | 助听器电池容量极小。**对策**：采用**事件驱动机制**（环境安静时 U55 休眠，仅用低功耗 VAD/传统算法；复杂场景唤醒 AI）及网络稀疏化。 |
| **双耳协同** | 空间声场对齐                                         | 单耳降噪会破坏空间定位。**对策**：双耳通过低功耗蓝牙（BLE Audio）进行微秒级同步，在片上进行**双耳融合（Binaural Fusion）**处理。 |

## 四、 M55/U55 异构架构落地部署策略



为了在 M55 + U55 硬件平台上实现实时流式处理，推荐采用以下异构流水线：

[带噪音频输入] │ ▼ ┌────────────────────────────────────────────────────────┐ │  Cortex-M55 (利用 Helium 向量指令集)                    │ │  • 音频重采样与加窗                                      │ │  • 高效硬加速短时傅里叶变换 (STFT)                       │ │  • 提取特征（如对数梅尔谱、复数特征）                     │ └─────────────────────────┬──────────────────────────────┘ │ (输入特征张量) ▼ ┌────────────────────────────────────────────────────────┐ │  Ethos-U55 微型 NPU (通过 Arm Vela 编译器优化)           │ │  • 执行纯神经网络推理 (INT8/INT16)                     │ │  • 硬件加速复数卷积/时间卷积 (TCN)                       │ │  • 输出高精度 Mask (掩蔽矩阵)                           │ └─────────────────────────┬──────────────────────────────┘ │ (预测的 Mask) ▼ ┌────────────────────────────────────────────────────────┐ │  Cortex-M55                                            │ │  • 将 Mask 应用回原频谱                                 │ │  • 逆短时傅里叶变换 (ISTFT)                             │ │  • 结合用户听力补偿曲线 (Late Fusion)                   │ └─────────────────────────┬──────────────────────────────┘ │ ▼ [降噪后高清晰度音频输出]

### 部署核心要点：

1.  **量化策略（Quantization）**：音频对量化噪声极为敏感。模型需通过 TensorFlow Lite Micro 进行全整数化。建议使用**量化感知训练（QAT）**，并对关键的 Mask 输出层采用 **INT16** 激活值以保证音质。
2.  **算子对齐（Operator Alignment）**：设计网络时，必须严格参考 Ethos-U55 支持的算子列表（Supported Operators）。尽量避免在网络中间使用复杂的非线性激活函数（如 Swish）。若出现不支持的算子，流水线将 Fallback 回 Cortex-M55 处理，这会造成额外的总线数据搬运开销。



## 声音降噪的 5 步标准流水线

```
[1. 采集与分帧] ──> [2. 时频转换 (STFT)] ──> [3. 特征提取与 AI 推理]
                                                     │
[5. 时域重构]   <── [4. 谱图滤波 (Apply Mask)] <────┘
```

### 1. 模拟采集与分帧（Frame Segmentation）

麦克风采集到的是连续的模拟声波，经过 ADC（模数转换器）变成一串数字采样点（例如助听器常用的 16kHz 采样率）。

- **切片（分帧）：** 算法无法一次性处理一整段音频，必须把连续的声音切成一小段一小段的“帧”（Frame）。
- **助听器的极端要求：** 普通蓝牙耳机通常采用 20ms 一帧，但助听器为了把总延迟控制在 8ms 以内，一帧通常只有 **2ms 或 4ms**（在 16kHz 采样率下，4ms 对应 64 个采样点）。
- **加窗（Windowing）：** 为了防止切片时边缘信号发生突变（产生人工杂音），会使用汉宁窗（Hanning）等数学窗函数让每帧的边缘平滑过渡。

### 2. 时频转换（STFT，短时傅里叶变换）

直接在时域（波形图）上很难分辨什么是噪音、什么是人声，因为它们全部混杂交织在一起。

- **为什么要转：** 声音是由不同频率的波组合而成的。通过短时傅里叶变换（STFT），算法把声音从**时域（时间-振幅）\**推向了\**频域（频率-能量）**。
- **结果：** 这一步把一帧帧的声音变成了一张**时频图（Spectrogram）**。图上的每个点，代表了“在某一个时间点，某一个频率上的声音有多大”。
- *在硬件中：这一步完全由 Cortex-M55 芯片利用 Helium 向量指令集在极短时间内硬件加速完成。*

### 3. 特征提取与 AI 推理（Feature Extraction & Inference）

这是降噪的“大脑”部分，也是 Ethos-U55 NPU 大显身手的地方。

- **提取特征：** 把时频图的数据进行简化（比如转成对数梅尔谱），作为输入喂给神经网络。
- **AI 算命（预测 Mask）：** 神经网络（如 Tiny-DCCRN 或 RNNoise）的任务不是直接去画一个纯净的声音，而是去预测一个“遮罩/掩蔽矩阵”（Mask）。
- **Mask 的含义：** 这个矩阵的维度和时频图一模一样，里面的数值通常在 $0.0 \sim 1.0$ 之间。$1.0$ 代表这个频率完全是人声，应该保留；$0.0$ 代表这个频率完全是噪音，应该抹去。

### 4. 谱图滤波（Apply Masking）

得到了 AI 预测的 Mask 之后，接下来的操作非常简单直接：

- **矩阵点乘：** 将原始带噪声音的时频图，与 AI 预测出的 Mask 矩阵进行**逐点相乘**。
- **效果：** 噪音所在的频率点因为乘了接近 $0$ 的数字，能量瞬间被压制消失了；人声所在的频率点因为乘了接近 $1$ 的数字，被完好地保留了下来。此时，原本被污染的时频图，就变成了一张“纯净人声的时频图”。

### 5. 时频逆变换与重构（ISTFT & OLA）

现在我们手里只有干净声音的“频谱图”，患者是没办法直接听图画的，必须把图再变回声波。

- **逆变换（ISTFT）：** 通过逆短时傅里叶变换，把滤波后的频域信号重新变回时域的一帧帧波形。
- **重叠相加（Overlap-Add, OLA）：** 把之前切开的、加了窗的一帧帧波形，按照切开时的比例重新“缝合/粘接”起来，消除帧与帧之间的断层。
- *助听器独有后处理：在声音送进耳放喇叭前，Cortex-M55 还会根据患者的个性化听力损失曲线（Audiogram），对不同频段进行针对性的增益放大（听力补偿）。*
